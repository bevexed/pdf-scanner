# 账单截图导出系统 · 实施文档

> 一个**本地离线**工具:财务把 PDF 账单喂进去,输入一批运单号,程序自动判断每个单号
> 有没有「杂费 / 关税」,把对应明细区域**截图 + 打码**后导出。财务电脑**零安装、不联网**。

---

## 1. 需求速览

### 1.1 两条业务线

| 业务 | 输入 | 判断逻辑 | 输出 |
|------|------|----------|------|
| **杂费账单** | 一批空运单号 AWB | 该单号在 PDF 中,除常规运费外**是否含杂费** | 有 → 截图明细并打码(发件人/运费/折扣/燃油/合计);无 → 返回「无杂费」 |
| **关税账单** | 一批运单号 | 该单号在 PDF 中**是否存在关税明细** | 有 → 截图关税明细并打码(寄件人信息);无 → 返回「无关税」 |

### 1.2 费用分类规则(杂费判定的核心)

- **常规运费(不算杂费)**:运费 / 折扣 / 燃油附加费 / Demand Surcharge
- **杂费**:超范围派送费 A、及以上之外的其他费用科目

> 判定方式:一票明细内,若出现「非常规运费」科目 → 判定为「有杂费」。
> 科目名称各期可能有中英文/简称差异,用一张**科目字典**维护(见 §5.3)。

### 1.3 数据库诉求(持续积累)

财务把所有 PDF 持续导入,程序建立本地索引(单号 → 文件+页码+坐标区域),
之后**只输单号即可查询**,喂得越多能查得越多。

### 1.4 关键前提(已确认)

- ✅ PDF 是**文字型**(可选中/复制文字)→ 无需 OCR,直接读文字坐标
- ✅ 各期账单**版式基本一致**→ 解析规则写一次长期可用
- ✅ 目标运行环境:**Windows**,财务**零安装、纯离线**

## 2. 技术方案

### 2.1 整体流程

```
导入 PDF(可批量、可累积)
  │
  └─→ 用 PyMuPDF 逐页读取所有文字块及其坐标(x0,y0,x1,y1)
        │
        └─→ 定位每个运单号的位置,切出「一票」的区域边界
              (从本单号所在行,到下一个单号出现之前)
              │
              └─→ 建索引:单号 → {文件, 页码, 区域矩形, 命中的费用科目}
                    │
                    └─→ 存入本地 SQLite 数据库(billing_index.db)

查询:输入一批单号
  │
  └─→ 查索引 → 命中?
        ├─ 是 → 判断有无杂费/关税
        │        ├─ 有 → 渲染该区域为图片 → 对指定字段坐标打码 → 导出 PNG
        │        └─ 无 → 返回「无杂费 / 无关税」
        └─ 否 → 返回「未在账单中找到」
```

### 2.2 技术栈

| 用途 | 库 | 说明 |
|------|----|----|
| 读 PDF 文字+坐标、渲染区域为图 | **PyMuPDF (fitz)** | 核心。文字型 PDF 能拿到精确坐标 |
| 图片裁剪 + 打码(模糊/马赛克) | **Pillow** | 对指定矩形区域做高斯模糊或色块覆盖 |
| 本地索引数据库 | **sqlite3**(Python 内置) | 无需额外安装 |
| 本地网页界面 | **Flask** + 原生 HTML/JS | 只监听 127.0.0.1,不出网 |
| 打包成 Windows exe | **PyInstaller** | 在 Windows 环境执行(见 §7) |

> 全部离线,无云端 AI、无网络请求。

### 2.3 交付形态

- 界面:**原生桌面窗口**(内嵌本机 127.0.0.1 服务,无终端黑窗、无需系统浏览器)
- 分发:**打包成单个 `.exe`**(Windows)/ `.app`(macOS),财务双击即用,无需装 Python / 任何依赖 / 联网

## 5. 核心模块实现要点

### 5.1 PDF 解析 `core/pdf_parser.py`

关键:用 PyMuPDF 拿到每个文字片段的坐标,再定位单号、切一票区域。

```python
import fitz  # PyMuPDF
import re

AWB_PATTERN = re.compile(r'\b\d{12}\b')   # 运单号:12 位数字(按实际调整)

def parse_pdf(pdf_path):
    """返回该 PDF 中所有命中的 {单号, 页码, 区域矩形, 页对象引用} 列表。"""
    doc = fitz.open(pdf_path)
    results = []
    for page_no in range(len(doc)):
        page = doc[page_no]
        words = page.get_text("words")  # [(x0,y0,x1,y1, "文字", block, line, word_no), ...]
        # 1) 找出所有单号 word 的位置
        awb_hits = [(w, w[4]) for w in words if AWB_PATTERN.fullmatch(w[4])]
        # 2) 对每个单号,按版式规则确定这一票的纵向边界:
        #    从本单号所在行 y0,到下一个单号所在行 y0(或页底/表尾)
        awb_hits_sorted = sorted(awb_hits, key=lambda t: t[0][1])  # 按 y 排序
        for i, (w, awb) in enumerate(awb_hits_sorted):
            y_top = w[1]
            y_bottom = (awb_hits_sorted[i+1][0][1]
                        if i + 1 < len(awb_hits_sorted) else page.rect.y1)
            rect = fitz.Rect(page.rect.x0, y_top - 4, page.rect.x1, y_bottom - 2)
            results.append({
                "awb": awb,
                "page_no": page_no,
                "rect": [rect.x0, rect.y0, rect.x1, rect.y1],
                "region_text": page.get_textbox(rect),  # 该区域内文字,供分类用
            })
    doc.close()
    return results
```

> ⚠️ **一票的边界规则**(y_top / y_bottom 怎么取)是最需要看样本 PDF 微调的地方。
> 多票混排、跨页时要专门处理。这是唯一需要真实样本才能定死的部分。

### 5.2 索引库 `core/indexer.py`

SQLite 表结构:

```sql
CREATE TABLE IF NOT EXISTS billing_index (
    id INTEGER PRIMARY KEY,
    awb TEXT,                 -- 单号
    pdf_file TEXT,            -- 来源 PDF 文件名
    page_no INTEGER,          -- 页码
    rect TEXT,                -- 区域矩形 JSON: [x0,y0,x1,y1]
    region_text TEXT,         -- 区域文字(供分类)
    bill_type TEXT            -- 'fee'(杂费类账单) / 'duty'(关税账单)
);
CREATE INDEX idx_awb ON billing_index(awb);
```

导入 PDF = 解析后批量 `INSERT`;查询 = `SELECT ... WHERE awb = ?`。

### 5.3 费用分类 `core/classifier.py`

```python
# 常规运费科目(命中这些不算杂费)
NORMAL_FEES = ["运费", "折扣", "燃油附加费", "Demand Surcharge", "fuel", "discount"]

def has_extra_fee(region_text: str) -> bool:
    """一票明细中,除常规运费外是否含其他费用科目 → 有杂费。"""
    lines = [ln.strip() for ln in region_text.splitlines() if ln.strip()]
    for ln in lines:
        looks_like_fee_row = _is_fee_row(ln)          # 该行是否是费用行(含金额)
        if looks_like_fee_row and not _is_normal(ln): # 是费用行但不属常规运费
            return True
    return False

def _is_normal(line: str) -> bool:
    return any(k.lower() in line.lower() for k in NORMAL_FEES)

# 关税判定:看区域文字里是否出现关税相关关键字/科目
def has_duty(region_text: str) -> bool:
    return any(k in region_text for k in ["关税", "Duty", "Customs"])
```

> 科目字典 `NORMAL_FEES` 放 `config.py`,随实际账单科目名扩充。

### 5.4 打码导出 `core/exporter.py`

```python
import fitz
from PIL import Image, ImageFilter
import io, os

# 需打码的字段(按业务):杂费单 → 发件人/运费/折扣/燃油/合计;关税单 → 寄件人
def export_masked(pdf_path, page_no, rect, mask_rects, out_path, zoom=3):
    """
    rect: 要截取的区域 [x0,y0,x1,y1]
    mask_rects: 该区域内需打码的子矩形列表(相对页坐标)
    """
    doc = fitz.open(pdf_path)
    page = doc[page_no]
    clip = fitz.Rect(*rect)
    mat = fitz.Matrix(zoom, zoom)  # 放大提高清晰度
    pix = page.get_pixmap(matrix=mat, clip=clip)
    img = Image.frombytes("RGB", (pix.width, pix.height), pix.samples)

    # 把页坐标里的打码矩形换算到裁剪图内的像素坐标,再做模糊
    for m in mask_rects:
        mx0 = int((m[0] - rect[0]) * zoom); my0 = int((m[1] - rect[1]) * zoom)
        mx1 = int((m[2] - rect[0]) * zoom); my1 = int((m[3] - rect[1]) * zoom)
        box = (max(0,mx0), max(0,my0), min(img.width,mx1), min(img.height,my1))
        if box[2] > box[0] and box[3] > box[1]:
            region = img.crop(box).filter(ImageFilter.GaussianBlur(12))
            img.paste(region, box)

    img.save(out_path)
    doc.close()
    return out_path
```

> **打码字段坐标怎么定位**:在解析阶段,除记录单号坐标外,同时记录"发件人""合计"等
> 字段标签的坐标,导出时对其对应值区域打码。这一步同样需要样本 PDF 校准坐标。

## 7. 打包成 Windows exe(关键步骤)

> ⚠️ **必须在 Windows 电脑(或 Windows 虚拟机 / 云主机)上执行。**
> Mac 无法交叉编译出 `.exe`。开发可在 Mac,打包这步换到 Windows。

### 方式 A:在 Windows 机器上直接打(推荐)

`build_win.bat`:

```bat
@echo off
python -m venv venv
call venv\Scripts\activate
pip install -r requirements.txt
pyinstaller --noconfirm --onefile --name 账单截图导出系统 ^
  --add-data "web;web" ^
  --hidden-import fitz ^
  app.py
echo 打包完成,exe 在 dist\ 目录下
pause
```

产物:`dist\账单截图导出系统.exe` → 直接发给财务双击运行。

### 方式 B:用 GitHub Actions 免费云端打包(你连 Windows 都不用碰)

在仓库加 `.github/workflows/build.yml`:

```yaml
name: build-windows-exe
on: [workflow_dispatch]
jobs:
  build:
    runs-on: windows-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-python@v5
        with: { python-version: '3.11' }
      - run: pip install -r requirements.txt
      - run: pyinstaller --onefile --name billing_exporter --add-data "web;web" --hidden-import fitz app.py
      - uses: actions/upload-artifact@v4
        with: { name: exe, path: dist/*.exe }
```

推代码后在 Actions 页面手动运行,下载生成的 `.exe`。

### 打包注意点

- `--add-data "web;web"`:把界面模板打进 exe(Windows 用 `;` 分隔,Mac/Linux 用 `:`)。
- `--hidden-import fitz`:PyMuPDF 有时需显式声明。
- `data/` 目录(数据库、导出图)让程序**运行时在 exe 同级自动创建**,不要打进 exe。
- 首次启动 exe 若被 Windows Defender 拦,右键"仍要运行"即可(自打包无签名,属正常)。
