# 账单截图导出系统 · 设计文档(Spec)

> 本地离线工具:财务把 FedEx 账单 PDF 导入,输入一批运单号,程序判断每单是否含
> 「杂费 / 关税」,把对应明细区域**截图 + 打码**后导出。财务电脑**零安装、纯离线、单 exe 双击即用**。

## 1. 技术选型(已定)

**Python 3.11 + PyMuPDF + Pillow + Flask + PyInstaller**,全离线,打包成单个 Windows exe。

选型依据(用户约束:开发快 & 质量稳 + 必须单 exe 双击):
- PyMuPDF 一个库同时搞定「读文字坐标 + 渲染区域成图」,对 5623 页大 PDF 性能与稳定性最佳。
- 实施文档已按 Python 给出七八成核心代码,落地最快。
- PyInstaller 打 Windows exe 是最成熟路径(Node 的 pkg 遇原生模块更折腾)。

| 用途 | 库 |
|------|----|
| 读 PDF 文字+坐标、渲染区域为图 | PyMuPDF (fitz) |
| 图片裁剪 + 打码(高斯模糊) | Pillow |
| 本地索引库 | sqlite3(内置) |
| 本地网页界面 | Flask + 原生 HTML/JS |
| 打包 Windows exe | PyInstaller |

## 3. 整体架构

```
财务网页上传 PDF(可批量、可累积)
  → 后台线程解析:PyMuPDF 逐页读文字+坐标
  → 定位每票(空运提单 AWB + 12位数字)+ 切区域 + 记录费用科目/寄件人/金额坐标
  → 判定有无杂费/关税(导入时预计算)
  → 写入 SQLite(billing_index.db),进度条实时更新
查询:粘贴一批单号 + 选「杂费/关税」
  → 查索引 → 读预计算的判定结果
  → 有:渲染区域→打码→导出 PNG
  → 无:「无杂费/无关税」;未命中:「未找到」
```

### 模块划分(单一职责,可独立测试)

| 文件 | 职责 | 依赖 |
|------|------|------|
| `core/config.py` | 科目字典、打码字段、版式参数(可编辑) | — |
| `core/pdf_parser.py` | 读坐标、切一票区域、记录科目和打码字段坐标 | PyMuPDF, config |
| `core/classifier.py` | 判定有无杂费/关税 | config |
| `core/indexer.py` | SQLite 读写(导入累积 + 按单号查) | sqlite3 |

| `web/` | 单页界面(导入/查询/结果/批量导出) | — |

目录结构沿用实施文档 §3。

## 4. PDF 解析 & 一票边界规则

**定位一票**:以 `空运提单 AWB` 标签锚定,取其**下一行**的 12 位数字为单号
(避开摘要页里同样的 12 位数字造成重复)。

**纵向边界**(文字坐标 y):
- `y_top` = 本票 `空运提单 AWB` 标签行 y0,往上留几像素(含序号/寄件日期)
- `y_bottom` = 本票 `合计 Total` 那行金额的 y1,往下留几像素;或下一个 `空运提单 AWB` 的 y0 —— **取先到者**
- 用「合计 Total」当下边界比「下一个单号」更准:合计后的「该票燃油附加费为…」「按体积重量…」说明行不属于本票明细,不截入。

**跨页处理**:一票通常在同一页内(已核实)。先按「单票不跨页」实现;若 AWB 在页底、合计在下页,单独标记并记日志,不做复杂跨页合并(YAGNI)。

**同时记录打码字段坐标**(基于真实坐标核实,三列布局):
- 寄件人区:`寄件人Sender` 标签(x≈243, y≈161)往下到 `运费` 标签行之前、x 在 243~395 的所有 word 包络框
- 金额值:费用标签(`运费/折扣/燃油/合计`,x≈300)所在行 y,取**同一行 x>530 的 `￥` word** 坐标(金额右对齐在 x≈533~564,不是标签下一行)

存进索引,导出时直接用。

**每票入库字段**:`{awb, pdf_file, page_no, rect, region_text, mask_rects, bill_type, has_extra, has_duty}`。

## 5. 费用分类

`config.py` 维护可编辑字典:
```python
NORMAL_FEES = ["运费", "Freight", "折扣", "Discount",
               "燃油附加费", "Fuel Surcharge", "Demand Surcharge"]
DUTY_KEYWORDS = ["关税", "Duty", "Customs", "海关"]
```

- **杂费判定(黑名单外全算杂费)**:逐条扫本票费用行,若某行是费用科目(标签+金额)
  但不在 `NORMAL_FEES` 白名单里 → 有杂费。新科目自动命中、不漏判。后续可改列表扩充。
- **关税判定**:本票文字含 `DUTY_KEYWORDS` 任一 → 有关税。
- **预计算**:判定在**导入时**完成并存进索引(`has_extra` / `has_duty`),查询时直接读。

## 6. 打码导出

**打码字段(已确认)**:
- 杂费单:寄件人 Sender + 运费/折扣/燃油金额 + 合计 Total 金额(收件人不打码)
- 关税单:寄件人 Sender(按实施文档)

**流程**:
1. `page.get_pixmap(clip=区域, matrix=Matrix(3,3))` 放大 3 倍渲染成清晰图
2. 打码矩形从页坐标换算到裁剪图内像素坐标
3. 每个矩形做 `GaussianBlur(12)` 覆盖回去
4. 导出 PNG 到 `data/exports/{awb}.png`

**边界防御**:打码矩形做 `max(0,..)/min(w,..)` 裁剪防越界;某字段坐标缺失时记日志跳过、不崩溃(宁可少打一个也不报错,导出后人工复核)。

## 7. 数据存储

```sql
CREATE TABLE IF NOT EXISTS billing_index (
  id INTEGER PRIMARY KEY,
  awb TEXT, pdf_file TEXT, page_no INTEGER,
  rect TEXT,              -- 区域矩形 JSON [x0,y0,x1,y1]
  region_text TEXT,
  mask_rects TEXT,        -- 打码矩形 JSON(寄件人/各金额)
  bill_type TEXT,         -- 'freight'
  has_extra INTEGER,      -- 有无杂费 0/1
  has_duty INTEGER        -- 有无关税 0/1
);
CREATE INDEX IF NOT EXISTS idx_awb ON billing_index(awb);

CREATE TABLE IF NOT EXISTS imported_files (
  pdf_file TEXT PRIMARY KEY, imported_at TEXT, ticket_count INTEGER
);
```
`imported_files` 记录已导入文件,按文件名+页数避免重复导入。

## 8. 界面(单页 index.html)

1. **导入区**:选/拖 PDF 上传 → 后台解析 → 进度条(轮询 `/progress`)→「已入库 N 票」
2. **查询区**:大文本框粘贴单号(每行一个)+ 单选「杂费 / 关税」+ 查询按钮
3. **结果表**:`单号 | 状态(有杂费✅/无杂费/未找到) | [预览][导出]`
4. **批量导出**:一键导出所有命中票的打码 PNG,打 zip 下载

**后台解析**:上传后开 `threading.Thread` 解析,主线程立即返回;前端轮询进度,5623 页不阻塞界面。

**接口**:`GET /`、`POST /import`、`GET /progress`、`POST /query`、`GET /preview/<awb>`、`POST /export`(批量,返回 zip)。服务仅监听 `127.0.0.1`,不出网。

## 11. 已确认的关键决策

| 决策点 | 结论 |
|--------|------|
| 技术栈 | Python + PyMuPDF + Pillow + Flask + PyInstaller |
| 分发 | 单 exe 双击,纯离线 |
| 杂费判定 | 黑名单(常规运费)外全算杂费,科目字典可编辑 |

| 跨页 | 先不做合并,标记记日志(YAGNI) |
| 判定时机 | 导入时预计算存索引 |
