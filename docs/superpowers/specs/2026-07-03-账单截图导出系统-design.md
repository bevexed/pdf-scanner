# 账单截图导出系统 · 设计文档(Spec)

> 本地离线工具:财务把 FedEx 账单 PDF 导入,输入一批运单号,程序判断每单是否含
> 「杂费 / 关税」,把对应明细区域**截图 + 打码**后导出。财务电脑**零安装、纯离线、单 exe 双击即用**。

## 1. 技术选型(已定)

**Python 3.11 + PyMuPDF + Pillow + Flask + PyInstaller**,全离线,打包成单个 Windows exe。

选型依据(用户约束:开发快 & 质量稳 + 必须单 exe 双击):
- PyMuPDF 一个库同时搞定「读文字坐标 + 渲染区域成图」,对 5623 页大 PDF 性能与稳定性最佳。
- 实施文档已按 Python 给出七八成核心代码,落地最快。
- PyInstaller 打 Windows exe 是最成熟路径(Node 的 pkg 遇原生模块更折腾)。

| 用途 | 库 |
|------|----|
| 读 PDF 文字+坐标、渲染区域为图 | PyMuPDF (fitz) |
| 图片裁剪 + 打码(高斯模糊) | Pillow |
| 本地索引库 | sqlite3(内置) |
| 本地网页界面 | Flask + 原生 HTML/JS |
| 打包 Windows exe | PyInstaller |

## 3. 整体架构

```
财务网页上传 PDF(可批量、可累积)
  → 后台线程解析:PyMuPDF 逐页读文字+坐标
  → 定位每票(空运提单 AWB + 12位数字)+ 切区域 + 记录费用科目/寄件人/金额坐标
  → 判定有无杂费/关税(导入时预计算)
  → 写入 SQLite(billing_index.db),进度条实时更新
查询:粘贴一批单号 + 选「杂费/关税」
  → 查索引 → 读预计算的判定结果
  → 有:渲染区域→打码→导出 PNG
  → 无:「无杂费/无关税」;未命中:「未找到」
```

### 模块划分(单一职责,可独立测试)

| 文件 | 职责 | 依赖 |
|------|------|------|
| `core/config.py` | 科目字典、打码字段、版式参数(可编辑) | — |
| `core/pdf_parser.py` | 读坐标、切一票区域、记录科目和打码字段坐标 | PyMuPDF, config |
| `core/classifier.py` | 判定有无杂费/关税 | config |
| `core/indexer.py` | SQLite 读写(导入累积 + 按单号查) | sqlite3 |
| `core/exporter.py` | 渲染区域→打码→导出 PNG | PyMuPDF, Pillow |

## 4. PDF 解析 & 一票边界规则

**只解析明细页**:仅解析文字含 `账单明细` 或 `BILLING DETAIL` 的页。摘要页(含 `账单摘要`)
每行是「序号 运费 客户账号 单号 日期 金额」挤在一行、**无 `空运提单AWB` 标签**,必须显式跳过,
不能依赖「标签恰好不存在」的运气。所有页码均为 **0 基**(PyMuPDF page index),写文档/测试一律 0 基。

**定位一票**:在明细页内,以 `空运提单 AWB` 标签锚定,取**同一行 x≈128** 的 12 位数字为单号。

**纵向边界**(文字坐标 y):
- `y_top` = 本票 `空运提单 AWB` 标签行 y0,往上留几像素(含序号/寄件日期)
- `y_bottom` = 本票 `合计 Total` 那行金额的 y1,往下留几像素;或下一个 `空运提单 AWB` 的 y0 —— **取先到者**
- 用「合计 Total」当下边界比「下一个单号」更准:合计后的「该票燃油附加费为…」「按体积重量…」说明行不属于本票明细,不截入。

**跨页处理**:一票通常在同一页内(已核实)。先按「单票不跨页」实现;若 AWB 在页底、合计在下页,单独标记并记日志,不做复杂跨页合并(YAGNI)。

**结构化提取费用行(不依赖 get_textbox 文本重组)**:parser 用 `words` 坐标提取每票的费用行,
每行 = `{label(科目名), amount_rect(￥坐标), amount_text}`。费用标签在 x≈300、金额 `￥` 在同行 x>530。
classifier **直接吃这个结构化列表**判定,不再把区域拼成文本再按行切(后者依赖 PyMuPDF 换行,脆弱)。

**同时记录打码字段坐标**(基于真实坐标核实,三列布局):
- 寄件人区:`寄件人Sender` 标签(x≈243, y≈161)往下到 `运费` 标签行之前、x 在 243~395 的所有 word 包络框
- 金额值:费用标签(`运费/折扣/燃油/合计`,x≈300)所在行 y,取**同一行 x>530 的 `￥` word** 坐标

存进索引,导出时直接用。

**每票入库字段**:`{awb, pdf_file, page_no, rect, region_text, mask_rects, bill_type, has_extra, has_duty}`。

## 5. 费用分类

`config.py` 维护可编辑字典:
```python
NORMAL_FEES = ["运费", "Freight", "折扣", "Discount",
               "燃油附加费", "Fuel Surcharge", "Demand Surcharge"]
DUTY_KEYWORDS = ["关税", "Duty", "Customs", "海关"]
```

- **杂费判定(黑名单外全算杂费)**:逐行扫,若某行下一行是纯金额(￥),该行即费用科目行;
  不在 `NORMAL_FEES` 白名单 → 有杂费。新科目自动命中、不漏判。后续可改列表扩充。
- **⚠️ 噪声行排除(p179-191 实测修正)**:合计是三行结构 `合计 Total` / `CNY` / `￥金额`,
  必须排除 `SKIP_LINES = ("合计","Total","CNY","USD","EUR","汇率","Conversion")` 行,
  否则 `CNY` 行(下一行是￥合计金额)会被误判为杂费科目 → 全票误判有杂费。
- **⚠️ VAT 陷阱**:关键词不能直接 `in text` 判断——p191 票 `889960603046` 收件人地址含
  `VAT NO: FI...`(客户税号,非费用)。因判定只吃**结构化费用行**(标签+金额),地址文字不参与,天然避开。
- **关税判定(绑定费用行,非全文扫)**:只有当**费用行的科目名**含 `DUTY_KEYWORDS`(如
  `多米尼加海关用户费 ... Customs User Fee`、`海关关税`)时才算有关税。避免地址/说明里的
  `Customs`/`海关` 误判。
- **预计算**:判定在**导入时**完成并存进索引(`has_extra` / `has_duty`),查询时直接读。

## 6. 打码导出

**打码字段(已确认)**:
- 杂费单:寄件人 Sender + 运费/折扣/燃油金额 + 合计 Total 金额(收件人不打码)
- 关税单:寄件人 Sender(按实施文档)

**打码方式(隐私优先)**:默认**实心不透明色块遮盖**(黑色矩形),无法反推;
`config.MASK_MODE` 可切 `"blur"`(高斯模糊,仅美观场景用)。

**流程**:
1. `page.get_pixmap(clip=区域, matrix=Matrix(3,3))` 放大 3 倍渲染成清晰图
2. 打码矩形从页坐标换算到裁剪图内像素坐标
3. 每个矩形按 `MASK_MODE` 遮盖(默认 `ImageDraw` 填黑,或高斯模糊)
4. 导出 PNG 到 `data/exports/{awb}.png`

**⚠️ 打码不完整则阻止导出**:导出前校验必打字段(寄件人 + 各金额)坐标是否齐全。
缺任一 → **不导出**,返回 `{status:"打码不完整", missing:[...]}`,由财务人工处理。
绝不静默导出漏打码的图(隐私事故风险)。边界防御:矩形做 `max(0,..)/min(w,..)` 裁剪防越界。

## 7. 数据存储

```sql
CREATE TABLE IF NOT EXISTS billing_index (
  id INTEGER PRIMARY KEY,
  awb TEXT,
  pdf_hash TEXT,           -- 来源 PDF 内容 SHA1(去重与溯源锚点)
  pdf_file TEXT,           -- 原始文件名(仅展示)
  invoice_no TEXT,         -- 账单号(来源账单,如 XXXXX0612)
  source_page_index INTEGER, -- 0基页码
  record_type TEXT,        -- 'detail'(明细,可导出)/ 'summary'(摘要,暂不解析)
  rect TEXT,               -- 区域矩形 JSON [x0,y0,x1,y1]
  fee_rows TEXT,           -- 结构化费用行 JSON [{label,amount_rect,amount_text}]
  mask_rects TEXT,         -- 打码矩形 JSON(寄件人/各金额)
  bill_type TEXT,          -- 'freight'
  has_extra INTEGER, has_duty INTEGER,
  import_batch_id INTEGER  -- 关联 import_batches
);
CREATE INDEX IF NOT EXISTS idx_awb ON billing_index(awb);

-- 导入批次(事务边界):导入前建 batch(status=pending),整批成功后置 committed
CREATE TABLE IF NOT EXISTS import_batches (
  id INTEGER PRIMARY KEY,
  pdf_hash TEXT, pdf_file TEXT, page_count INTEGER,
  imported_at TEXT, ticket_count INTEGER,
  status TEXT              -- 'pending' / 'committed'
);
CREATE INDEX IF NOT EXISTS idx_batch_hash ON import_batches(pdf_hash);
```

**去重(按内容 hash,不靠文件名)**:导入前算 `pdf_hash`(SHA1)。若已存在
`status='committed'` 的同 hash 批次 → 跳过。同名不同内容不会误判已导入,改内容即重算。

**事务/回滚**:导入前(1)删除同 hash 的旧 `pending` 批次及其 `billing_index` 记录(清半成品);
(2)建新 `pending` 批次,分批 INSERT 时带 `import_batch_id`;(3)全部页解析完才把批次置
`committed`。查询只读 `committed` 批次的记录 → 半途失败的数据永不可见。

**只解析明细**:摘要页(`record_type` 会是 summary)当前**不写库**(YAGNI),仅解析 detail 页;
`record_type` 字段保留,为将来摘要查询留位。

## 8. 界面(单页 index.html)

1. **导入区**:选/拖 PDF 上传 → 后台解析 → 进度条(轮询 `/progress`)→「已入库 N 票」
2. **查询区**:大文本框粘贴单号(每行一个)+ 单选「杂费 / 关税」+ 查询按钮
3. **结果表**:`单号 | 来源账单号 | 页码 | 状态(有杂费✅/无杂费/未找到) | [预览][导出]`
   - 同一 AWB 命中多条(不同账期/重复导入)→ **每条一行**都列出,显示来源账单号供财务区分
4. **批量导出**:一键导出所有命中票的打码 PNG,打 zip 下载。打码不完整的票标红「打码不完整」,不导出

**后台解析**:上传后开 `threading.Thread` 解析,主线程立即返回;前端轮询进度,5623 页不阻塞界面。

**接口**:`GET /`、`POST /import`、`GET /progress`、`POST /query`、`GET /preview/<awb>`、`POST /export`(批量,返回 zip)。服务仅监听 `127.0.0.1`,不出网。

## 9. 打包(Windows)

- `build_win.bat`:PyInstaller `--onefile --add-data "web;web" --hidden-import fitz`
- **`--onefile` 路径拆分(关键)**:资源(web 模板/静态)在解压临时目录 `sys._MEIPASS`;
  运行数据必须在 **exe 真实所在目录**旁,否则写进临时目录、退出即丢:
  - 资源路径:`RESOURCE_DIR = sys._MEIPASS if getattr(sys,'frozen',False) else 项目根`
  - 数据路径:`RUN_DIR = os.path.dirname(sys.executable) if frozen else 项目根`,`DATA_DIR = RUN_DIR/data`
  - Flask 的 `template_folder/static_folder` 用 `RESOURCE_DIR/web`
- **⚠️ GitHub Actions 与「数据敏感/离线」冲突**:CI 仅用于**不含任何真实账单 PDF 的源码仓库**,
  绝不把 `var/`、`data/pdfs/`、样本 PDF 推上远程。`.gitignore` 必须排除。CI 只编译源码产出 exe。
- `data/`(数据库、导出图)运行时在 exe 同级自动创建,不打进 exe

| 决策点 | 结论 |
|--------|------|
| 技术栈 | Python + PyMuPDF + Pillow + Flask + PyInstaller |

| 跨页 | 先不做合并,标记记日志(YAGNI) |
